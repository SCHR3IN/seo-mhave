# Задача M001: Оптимизация воронки отслеживания электронной коммерции (E-commerce)

## 🎯 Цель задачи
Устранить потери воронки «карточка → корзина → оформление → покупка» путем внедрения сквозного отслеживания Enhanced Ecommerce, исправления JS-ошибок при формировании `dataLayer` и устранения интерфейсных барьеров на странице оформления заказа.

---

## 🛠️ Реализация и технические изменения

### 1. Активация Enhanced Ecommerce в компоненте оформления заказа
В файле `/order/index.php` (настройки компонента `bitrix:sale.order.ajax`) были изменены следующие параметры:
```php
"USE_ENHANCED_ECOMMERCE" => "Y",
"DATA_LAYER_NAME" => "dataLayer",
"BRAND_PROPERTY" => "BRAND",
```
Это позволило Битриксу автоматически формировать данные о корзине и товарах и пушить их в массив `window.dataLayer`.

### 2. Исправление JS-ошибки затенения переменной в `order_ajax.js`
В функции `setAnalyticsDataLayer` происходил сбой из-за повторного использования переменной `i` во внутреннем цикле, которая перетирала индекс внешнего цикла:
```javascript
// Было (ошибка):
for (var i in order.PROPS) { ... }

// Стало (исправлено):
for (var k in order.PROPS) { ... }
```
Этот сбой приводил к прерыванию выполнения скрипта и блокировал отправку транзакции в Метрику.

### 3. Устранение интерфейсных барьеров (UX)
В ходе аудита были выявлены и исправлены две проблемы:
* **Некорректный ввод телефона**: Настроен жесткий плейсхолдер телефона `+7 (999) 999-99-99` вместо пустого поля, чтобы исключить ошибки формата при вводе.
* **Случайные клики по отправке заказа**: Кнопка «Оформить заказ» теперь заблокирована (`disabled`) и имеет полупрозрачный вид, пока пользователь не установит галочки согласия с Политикой конфиденциальности и Условиями оферты. При установке галочек кнопка активируется автоматически.

## 🔄 Новые задачи и рекомендации, выявленные в процессе
1. **Кэширование скриптов**: После любых изменений в `order_ajax.js` необходимо обязательно сбрасывать объединенный кэш Битрикса в админ-панели (`/bitrix/admin/cache.php`), так как старые версии скрипта могут кэшироваться на стороне CDN или браузера пользователя.
2. **Синхронизация полей**: Рекомендуется добавить аналогичную валидацию оферты на мобильную версию корзины/оформления, если шаблоны разделены.
