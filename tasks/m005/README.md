# Задача M005: Исправить soft 404 страницы московского магазина

## 🎯 Цель задачи
Устранить Soft 404 ошибки на сайте для адресов, связанных со старыми/закрытыми магазинами (в частности `/contacts/stores/` и `/contacts/stores/moskva/404/`). Эти страницы должны возвращать реальный HTTP-код 404 (Not Found), быть исключены из карты сайта (`sitemap.xml`) и навигации, а ссылки с них перенаправлены на актуальные страницы.

---

## 🛠️ Реализация и технические изменения

### 1. Настройка принудительной отдачи 404 в Битрикс
В файле `/contacts/stores/index.php` на боевом сервере добавлена инициализация ядра Bitrix и принудительная отправка HTTP-заголовков 404 ошибки перед выводом контента:
```php
CHTTP::SetStatus("404 Not Found");
@define("ERROR_404","Y");
```
Это гарантирует, что при обращении к несуществующим адресам в разделе магазинов сервер корректно сообщает поисковым роботам о том, что страница не найдена.

### 2. Очистка навигационных меню
Ссылки на старый раздел магазинов `/contacts/stores/` заменены на основную страницу контактов `/contacts/` в меню сайта:
* В нижнем меню: `.bottom5.menu.php`
* В левом меню инфо-раздела: `info/.left.menu.php`

### 3. Исправление путей в каталоге
В настройках комплексного компонента каталога в `/catalog/index.php` параметр путей к складам/магазинам `STORE_PATH` изменен с `"/contacts/stores/#store_id#/"` на `"/contacts/"` для предотвращения генерации битых внутренних ссылок в карточках товаров.

### 4. Обновление карты сайта
Запущена генерация и постобработка `sitemap.xml`. Настроенные фильтры очистки успешно исключили адреса `/contacts/stores/` из итогового файла карты сайта.
